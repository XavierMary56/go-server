# API 接口性能分析与优化建议

适用场景：聚焦接口主链路延迟、响应体大小、下游依赖耗时与接口级吞吐问题。仓库命令、测试与通用行为约束应以 `repo-go-guardrails.md` 和 `CLAUDE.md` 为准，不在本文件中重复定义。

## 一、重点分析方向
- 单接口 RT 与 P95/P99
- 是否存在多次下游调用串行化
- 返回结构是否过大
- 鉴权、日志、序列化是否占比过高

## 二、优化建议
- 缩短主链路，只保留必要逻辑
- 并行无依赖下游请求
- 精简响应字段
- 热点读接口走缓存
- 对非强一致逻辑做异步化

## 三、审查清单
- 是否循环调用 RPC/SQL
- 是否每次请求做重复鉴权解析
- 是否返回了前端不用的大字段
- 是否未设置超时与熔断

## 四、关键指标
- RT / P95 / P99
- QPS
- 错误率
- 下游依赖耗时分布
